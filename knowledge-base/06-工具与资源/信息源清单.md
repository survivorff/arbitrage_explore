# 信息源清单

> 站在信息前端的前提，是有一套**分层的信息源矩阵**。本清单是框架模板——按 `01-认知基础/04-信息差` 的"信息源地图"分层，请根据你深耕的赛道，填入具体的源。
>
> ⚠️ 原则：**往上游（①②层）走，对下游（⑤层）保持警惕。** 信息源宁缺毋滥，定期淘汰噪音源。

---

## 一、信息源分层框架（回顾）

```
①一手参与者  ②专业社区  ③垂直媒体  ④聚合平台  ⑤大众媒体
 最早最准 ────────────────────────────────→ 最晚（已收敛）
```

每个赛道都应该尽量覆盖①②③层 + 必要的④层数据看板。

## 三、信息采集机制（让信息主动来找你）

- **RSS / 订阅**：把能订阅的源接入 RSS 阅读器，集中查看。
- **关键词告警**：对关键事件设置关键词推送。
- **Bot / 自动监控**：对可量化的指标（价差、价格、上新）写监控脚本/bot 告警。
- **人脉网络**：①层一手信息往往来自人，而非公开渠道。有意识地结交圈内人。

> 这部分的自动化，正是你第二步"工程产品"可以落地的方向之一（机会扫描/监控系统）。

---

## 四、信息源维护原则

1. **定期体检**（月复盘时）：哪些源持续给价值？哪些是噪音？砍掉噪音。
2. **持续上移**：努力从③④层往①②层渗透，那里才有真正的信息差。
3. **交叉验证**：重要信息至少两个独立源验证，防假消息/被收割。
4. **区分一手和二手**：明确标注每个源的"新鲜度层级"，对二手信息打折看待。
