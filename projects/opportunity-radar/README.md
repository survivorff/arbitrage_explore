# 机会雷达（Opportunity Radar）

> 工作名，可改。一个**信息差机会订阅服务**：把散落在各赛道源头的一手信号，加工成可直接行动的机会情报，订阅交付给愿意为"省去信息采集+判断"付费的人。
>
> 本质：这是 `knowledge-base` 里 **C 赛道（信息/知识套利）** 的落地项目，也是把整个知识库"产品化"的载体。

---

## 一句话定位

> **"我替你站在信息前端，把原始噪音加工成结构化的机会判断——你不用自己刷一整天，就能拿到经过框架评估、含风险提示的机会清单。"**

不是喊单，不是代客理财，是**信息 + 研究 + 方法论**的订阅。

---

## 这个项目在套利什么（信息差闭环）

```
源头原始信号（散、杂、噪音多、要花大量时间）
        │  ← 你用框架+信息源网络+持续投入，做了"加工"
        ▼
结构化机会情报（已评估、含风险、可直接判断是否行动）
        │  ← 订阅者愿意为"省下的时间+认知差"付费
        ▼
订阅收入（一份加工，卖给 N 个人，边际成本近零）
```

- **价差来源**：③信息（你更早/更全）+ ④形态加工（你把噪音变成判断）+ ⑦规模（一份内容卖多份）。
- **护城河**：持续产出能力 + 真实可验证的战绩 + 独家方法论（即本知识库）+ 社区网络效应。
- **最大风险**：合规（投顾红线）+ 信誉（一次造假全盘崩）。见 `06-合规与信任框架.md`。

## 阅读顺序

1. 先读 `01` 想清楚"卖给谁、凭什么"。
2. 再读 `02`+`03`+`04` 想清楚"机会从哪来、怎么加工"（信息采集系统）。
3. 然后 `05` 想清楚"怎么交付和收费"。
4. `06` 全程必须遵守的红线。
5. `07` 落地：现在就能做的第一步。
