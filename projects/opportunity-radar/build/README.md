# 机会雷达 · 立项设计（build）

> 本目录是机会雷达项目的**正式立项设计文档**，按 `市场调研 → 商业模式 → 运营 → 产品设计 → 技术开发` 五步推进。
> 上层 `../01~07` 是项目的**概念设计**（通用框架），本目录是**落地立项**（锁定决策后的具体方案）。

## 锁定的项目前提

| 维度 | 决策 |
|------|------|
| 目标市场 | 国内为主，海外为辅（阶段2+） |
| 起步赛道 | AI 工具 / 红利 |
| 投入 | 高投入（每天可投） |
| 技术 | 有能力，自建工具 |
| 变现 | 先养+验证，后收费（多元变现） |
| 获客阵地 | 公众号（沉淀）+ 小红书（拉新） |
| 付费载体 | 知识星球（起步） |

## 五步文档

| 步骤 | 文档 | 核心结论 |
|------|------|---------|
| 1 | `01-市场调研.md` | 需求已验证；变现要多元（非纯订阅）；国内走公众号+星球；差异化在"判断层" |
| 2 | `02-商业模式.md` | 多元分阶段变现；成本极轻；护城河靠叠加；最大风险=信任+精力 |
| 3 | `03-运营.md` | 内容驱动的信任飞轮；小红书拉新+公众号沉淀；质量>数量；阶段目标 |
| 4 | `04-产品设计.md` | 两层产品（内容+工具）；机会卡片是数据原子；工具MVP=采集+初筛+评估 |
| 5 | `05-技术开发.md` | Python+SQLite+Streamlit；两周跑通MVP最小闭环；人机分工铁律 |
