# 立项 · 第六篇：机会的定义与数据来源（核心思考）

> 这一篇是产品的"灵魂层"。前面五步定的是流程和工具，这一篇定的是**"什么是值得我们追踪的机会"**——
> 这个问题想清楚了，数据源、信号类型、初筛规则都自然推出来。
>
> 写于完成 MVP 后。一边跑数据一边想清楚的事。

---

## 一、为什么这个问题必须重新想

之前的设计里，"机会"被隐含定义成两类：
1. **资讯型**（AI 赛道）：发布、限免、降价、新工具——以"文章"形式存在。
2. **数据型**（加密赛道）：APY、资金费率、趋势——以"数字"形式存在。

但跑过一轮后，会发现这个二分法**太粗**：
- 资讯型里大量是"广告/八卦/无关报道"，真正"机会"的密度低。
- 数据型里 Binance 资金费率 -2000% 的极端值看起来诱人，**但很可能是新币、低流动性、根本做不了**——这是"伪机会"。
- 收件箱被这些填满，反而会稀释你的判断力。

> **核心问题：什么样的信号才算"值得用户行动的机会"？什么样只是"信息"或"噪音"？**

---

## 二、机会的第一性定义（贯穿所有赛道）

回到知识库 `01-认知基础`：

> **机会 = 一个"价值在某维度上存在价差，且价差 > 摩擦"的、当前可被特定群体捕捉的具体行动。**

抽象出三要素：

```
①价差（Spread）        — 这件事到底有没有"低买高卖"的空间？
②摩擦/可达性（Reach）  — 我们的受众有能力 / 资源 / 资格去做吗？
③时效（Window）        — 这个空间能存在多久？需要多快？
```

**只满足"有信息"不算机会。** 比如"OpenAI 发布新模型"——这是信息，不是机会，除非你能把它转化成"普通人现在能拿到什么红利、能省多少钱、能用它做什么"。

---

## 三、机会的分级（这是我们应该往工具里落的）

按"对用户的可行动性"分四级。我们要在工具里**显式标注**信号属于哪一级，让收件箱不再是平铺的列表，而是按"机会成熟度"分层。

### 🟢 L4 · 即时可行动机会 (Actionable Now)
> 用户**今天/这周**就能直接去做、且摩擦清晰、收益可估算的机会。

例：
- "X 工具限免 7 天，注册即得 100 美元额度" → 立刻注册
- "Y 池子 stETH-USDC 稳定币 18% APY，TVL $50M，无 IL 风险" → 可以参与
- "Z 协议空投快照在 3 天后，参与门槛 = 桥 0.1 ETH 过去并交互一次" → 可以行动

特点：**有明确动作、有时效、价差大于摩擦、风险可估**。这是简报的主角，订阅价值最高。

### 🟡 L3 · 待评估机会 (Worth Evaluating)
> 看起来像机会，但**需要进一步研究**才能判断值不值得做。

例：
- "新公链空投积分活动启动" → 是真红利还是太多人参与摊薄？要看进度。
- "某资金费率连续高位 3 天" → 是结构性还是短期情绪？要看是否可持续。
- "新 AI 工具发布" → 是真好用还是噱头？要试。

特点：**有潜力，但要研究**。这一级是"你提供研究价值"的核心区——读者自己判断不了，看你的判断。

### 🔵 L2 · 趋势/背景信息 (Trend / Context)
> 不是直接机会，但**形成判断不能不知道**。

例：
- "美联储降息 25bp" → 不是单笔机会，但影响所有加密、AI、跨境
- "Anthropic 估值 1500 亿美元" → 行业风向
- "国内 AI 政策更新" → 影响赛道

特点：**塑造判断框架**。可以做月度复盘，但不该填满每周收件箱。

### 🔴 L1 · 噪音 (Noise)
> 应该被过滤掉。

例：
- 招聘、八卦、营销稿、重复转发、与受众无关
- 已收敛的（满天飞的，谁都看到过的）
- 假机会（涉嫌骗局、涉灰）

特点：**直接屏蔽**。比"误把噪音当机会"重要得多——一条骗局信号污染你的口碑。

## 四、机会信号的"标准元数据"（落到数据模型）

每条信号在被人评估前，应该尽可能机器化地标注：

```
信号 = {
  basics:      {title, source, time, url}                   ← 已有
  spread:      {what_kind, magnitude, direction}            ← 价差是什么、多大、方向
  friction:    {action_needed, capital_min, complexity}     ← 要做什么动作、最小资金、难度
  window:      {urgency, expires_at, repeatable}            ← 多急、多久过期、可重复吗
  fit:         {target_user, region, requires_kyc}          ← 适合谁、地域、是否需要KYC
  risk:        {category, severity, scam_flag}              ← 风险类别、严重度、是否骗局
  level:       L1/L2/L3/L4                                  ← 机会成熟度
}
```

> 这是**"机会的结构化定义"**。我们工具的进化方向就是：让机器尽可能填好这些字段，人只在 L3/L4 上做最终评估。

## 五、由"机会定义"反推数据来源（关键）

不同赛道，**捕捉这四级机会需要的数据源结构完全不同**。这是回答你"数据来源应该不同"问题的本质。

### 通用模式

| 机会等级 | 信号特征 | 数据来源类型 |
|---------|---------|-------------|
| 🟢 L4 即时可行动 | 带明确数字+条件+时效 | **API/链上/官方公告** —— 一手且结构化 |
| 🟡 L3 待评估 | 有事实+需要判断 | 社区讨论 + 关键人物 + 媒体深度 |
| 🔵 L2 趋势/背景 | 大势/政策/资本动向 | 行业报告 + 数据看板 + 政策原文 |
| 🔴 L1 噪音 | 各类杂讯 | 自动过滤掉 |

### 推论：好的赛道数据源 = "L4 优先 + L3 兜底 + L2 月度 + L1 严过滤"

意味着每个赛道我们要为它配齐：
1. **结构化数据 API**（产 L4） — 最难找、最稀缺、最值钱。
2. **社区/关键人物源**（产 L3） — 一手判断的来源。
3. **政策/数据看板源**（产 L2） — 形成框架。
4. **过滤规则**（剔除 L1）。

### 现在工具的现状对照

| 赛道 | L4 来源 | L3 来源 | L2 来源 | L1 过滤 |
|------|--------|--------|--------|---------|
| 加密Web3 | ✅ DefiLlama/资金费率 | ⚠️ 弱（只有 RSS 媒体） | ❌ 缺 | ⚠️ 待加强 |
| AI工具 | ⚠️ 弱（限免/降价多在社媒，不在RSS） | ⚠️ 弱 | ⚠️ 一般 | ⚠️ 一般 |

> **结论：现在的工具能产出 L4（加密的链上数据） + 一些 L2（媒体），但 L3（社区判断）和 L1 过滤都很弱。这是下一阶段最该补的能力。**

## 六、新赛道的"机会蓝图"（你扩展赛道的指引）

每加一个赛道，先回答这五个问题：

1. **这个赛道的"价差"通常以什么形式存在？**（数字/事件/政策/能力差）
2. **L4 即时可行动机会，长什么样？怎么用一句话描述？**（几个例子）
3. **L4 信号的"权威源头"在哪？是不是 API/官方/链上数据？**
4. **L3 判断需要去哪些社区/关键人物处采集？**
5. **谁是受众？能不能 KYC/有资金/有技术能做？**（影响 fit 字段）

回答清楚再加，否则只是搬运噪音。

### 示例：跨境电商赛道（仅作蓝图示范，未实现）
1. 价差 = 海外/国内同款商品价差、平台费率差、汇率差。
2. L4 例 = "Amazon US 某品类批量进货价 vs 拼多多卖价利差 25%"。
3. L4 源 = 选品工具 API（卖家精灵、Helium10）、亚马逊价格 API、汇率 API。
4. L3 源 = 卖家社群、运营博主、平台政策动态。
5. 受众 = 跨境电商卖家。

## 七、对工具的下一步要求（行动清单）

基于上面的分析，工具该做这些：

- [ ] **数据模型扩展**：signals 增加 `level`(L1-L4) 和 `scam_flag` 字段。
- [ ] **L4 标注规则**：给数据采集器（如收益率/资金费率）补充阈值和过滤，标 L4 vs L2。
- [ ] **L3 来源接入**：扩展采集器到 Twitter/X 关键账号、社区精华——这是下一阶段重点。
- [ ] **L1 噪音过滤**：扩充黑名单（招聘/八卦/纯融资）+ 可选 AI 二筛。
- [ ] **欺诈/合规预警**：对加密的"年化几千%"自动加 ⚠️ 标注，对涉嫌骗局的关键词预警。
- [ ] **赛道蓝图模板**：新增赛道时，强制走"五问蓝图"，把每个赛道的 L1-L4 来源画清楚再写代码。

## 八、一句话总结

> **机会不是"信息"，是"对特定人在特定时间窗口内、价差大于摩擦的可行动点"。**
> 我们的差异化不在"采集得快"，在"分得清 L1-L4"——尤其在 L4（精准）和 L1（不污染）两端做到位。
> 数据源永远服务于"你想抓哪一级机会"——没想清楚机会等级，谈数据源就是空谈。
