## 这个产品到底在做什么？（一句话）

> 它帮你把**散落在各处的原始信号**，加工成**可行动的机会判断**。
> 本质是信息差套利：你比别人更早、更准地发现机会，省下他们的信息采集时间——这就是你能变现的价值。

```
数据源 → [机器]采集+初筛 → 📥收件箱 → [你]判断评估 → 🎯机会卡片 → 📰简报 → 发布变现
        机器干脏活累活               你只做最值钱的"判断"
```

---

## 为什么加密赛道的数据源和 AI 赛道不一样？

这是你之前的关键疑问。答案：**不同赛道的机会，藏在不同地方**。

| 赛道 | 机会长什么样 | 数据源类型 | 具体来源 |
|------|------------|-----------|---------|
| **加密Web3** | 带数字的量化机会：APY、资金费率、趋势 | **数据 API** | DeFiLlama、Binance、CoinGecko |
| AI工具 | 文章/发布：新工具、限免、降价 | 资讯 RSS | 厂商博客、社区、媒体 |

> 加密的钱不在新闻里，在**链上数据**里。所以加密采集器抓的是：
> - 💰 **DeFi 收益率**（哪个池子 APY 异常高 = 收益机会）
> - 📈 **资金费率**（哪个合约费率极端 = delta-neutral 套利机会）
> - 🔥 **趋势币**（哪些币关注度异动）
> - 📰 交易所/媒体公告（新币上线等，RSS 补充）

## 跟着走一遍（命令行 3 条 + 界面 5 步）

### 准备：启动

```bash
cd projects/opportunity-radar/engine
# 用仓库根目录的虚拟环境
../../../.venv/bin/python radar.py init          # 初始化数据库
../../../.venv/bin/python radar.py setup crypto  # 准备加密赛道数据源
../../../.venv/bin/python radar.py scan crypto   # 采集！（拉真实收益率/资金费率/趋势+资讯）
../../../.venv/bin/python radar.py ui            # 启动界面
```

> `scan crypto` 跑完会告诉你新增了多少条信号（通常 150+ 条真实机会信号）。

### 界面操作：5 步闭环

**第 1 步 · 🏁 使用引导** → 打开界面默认就是引导页，再读一遍流程。

**第 2 步 · 📥 收件箱** → 这里是机器筛好的信号，按相关度排序。
- 顶部可按**赛道/相关度/关键词/来源**筛选（试试搜"APY"或"资金费率"）。
- 加密信号会显示 **📊 关键数字**（如 `APY %: 48.36`）。
- 看到值得做成内容的机会 → 点 **「⬆️ 晋升为机会」**；噪音 → **「🗑️ 过滤掉」**。

**第 3 步 · 🎯 机会卡片** → 对晋升的机会做评估：
- 选套利维度、填"是什么/为什么值得关注/风险/适合谁"。
- 用**六维框架打分**（净价差/资金效率/规模/护城河/风险/合规）。
- ⚠️ 风险或合规打 1 分会触发"一票否决"——这是防止你推荐高风险垃圾的护栏。
- 写下你的判断（关注/观望/跳过）和理由 → 保存 → 标记发布。

**第 4 步 · 📰 简报生成** → 勾选评估好的机会 → 一键导出 Markdown 简报。
- 简报含速览、详情、风险、免责声明，人工润色后即可发公众号/小红书/社群。

**第 5 步 · 📈 战绩追踪** → 过几天回来，记录"这个机会后来怎么样了"。
- 诚实记录命中和**看错的** → 这是你最重要的信任资产。

## 一个具体例子（加密资金费率套利）

1. `scan crypto` 后，收件箱出现：`📈 [资金费率] XXXUSDT 年化 -300%（空头付费，可做多收费）`
2. 你判断：这是一个 delta-neutral 套利信号（现货做多 + 合约做多吃费率，对冲方向风险）。
3. 晋升 → 评估：净价差中、资金效率高、风险中（爆仓+费率反转）、合规需注意。
4. 写judgment："关注，但提示爆仓和费率反转风险，适合有合约经验的人"。
5. 生成简报 → 发布。这就是一条**有判断、含风险、不喊单**的机会情报。

> 注意：极端高的资金费率（如年化几百上千%）往往伴随高风险（新币、低流动性），
> 工具把它呈现出来，但**判断和风险提示是你的价值**——这正是产品的核心。

## 常见问题

**Q：数据会自动更新吗？**
A：MVP 是手动 `scan`（或界面点「立即扫描」）。资金费率/收益率每次扫描都是最新的。

**Q：能加别的赛道吗？**
A：能。在 `tracks.py` 注册一个新赛道，声明它的采集器即可。架构就是为"不同赛道不同源"设计的。

**Q：AI 初筛怎么开？**
A：复制 `.env.example` 为 `.env`，填 OpenAI 兼容的 key，设 `RADAR_AI_ENABLED=true`。不开也能用（规则初筛兜底）。

**Q：这些数据源要钱吗？**
A：加密三个 API（DeFiLlama/Binance/CoinGecko）都免费、无需 key。
