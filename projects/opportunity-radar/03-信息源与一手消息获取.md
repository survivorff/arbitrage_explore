# 03 · 信息源与一手消息获取

> 这是整个项目的**发动机**。你的产品质量，上限就是你的信息源质量。
> 核心目标：**比订阅者更早、更全、更准地拿到信号——并且尽量往源头①②层走。**

---

## 一、先理解：信息差的三层 × 信息源的五层

回顾知识库 `01-认知基础/04-信息差`：

- 信息价值三层：**知道 < 理解 < 能力**。你要交付的是"理解"层（已加工的判断），靠的是你处在"能力"层。
- 信息源五层（离源头远近）：

```
①一手参与者 → ②专业社区 → ③垂直媒体 → ④聚合平台 → ⑤大众媒体
 最早最准 ───────────────────────────────────→ 最晚（已收敛）
```

> **铁律：你的竞争力来自①②层。** 如果你只看④⑤层（聚合站、热搜），那你和订阅者拿到的是同样的二手信息，没有价差，没人会付费。

---

## 二、信息源矩阵设计（按层级 × 渠道类型）

为你选定的赛道，按下面的矩阵系统化地建源。**目标是每个赛道都覆盖①②层 + 必要的④层数据。**

### 通道类型总览

| 通道类型 | 典型平台 | 处于哪层 | 拿到的信息特点 |
|---------|---------|---------|--------------|
| **官方一手** | 项目方/厂商官网、官方博客、官方社媒、官方公告频道 | ① | 最准、最早的"事实" |
| **核心社区** | Discord、Telegram 核心群、Slack、专业论坛 | ②| 早期讨论、圈内 alpha |
| **代码/产品源** | GitHub（commit/release/issue）、产品更新日志、应用商店更新 | ① | 比公告还早（代码先行） |
| **数据/链上** | 数据看板、链上浏览器、监控工具、API | ①数据 | 客观、可量化的异动 |
| **关键人物** | 创始人/核心开发者/资深从业者的个人账号 | ①② | 一手观点和风向 |
| **垂直媒体/newsletter** | 行业资讯站、优质 newsletter、分析师 | ③ | 已加工，补充理解层 |
| **聚合/榜单** | 聚合站、榜单、趋势页 | ④ | 方便但人人可见 |
| **大众/热搜** | 微博/抖音热搜、大众新闻 | ⑤ | 多为离场信号 |

> 把每个赛道的具体源，逐条填进 `templates/信息源登记表.md`，并标注它属于哪一层、新鲜度如何。

## 三、如何拿到"一手消息"（核心方法）

"一手"不是靠运气，是靠**主动布局 + 自动化监控 + 人脉网络**。五个方法：

### 方法 1：盯住"事实的发源地"，而不是"转述的地方"
信息一定有一个最早出现的地方。倒推它：
- 一个产品要发布 → 最早出现在：官方博客/官方社媒/GitHub release/应用商店更新/官方社区公告。
- 一个政策要变 → 最早出现在：官方公告原文，而非媒体解读。
> **直接订阅"发源地"，而不是等媒体转述。** 媒体转述时，价差已经在收敛了。

### 方法 2：让信息主动来找你（自动化监控）
不要靠"手动刷"，靠"推送"。把可订阅的源接成自动监控：
- **RSS**：把支持 RSS 的官方博客、newsletter、论坛接入一个 RSS 阅读器，集中查看。
- **关键词告警**：对关键事件设关键词监控（社交平台搜索、Google Alerts 类工具）。
- **GitHub 监控**：Watch 关键仓库的 release/commit；用 GitHub 的 release feed。
- **频道转发/聚合 bot**：把多个 Telegram/Discord 频道的消息聚合到一个地方（注意合规和频道规则）。
- **数据告警**：对可量化指标（价格、上新、资金流、费率）写监控脚本/用现成告警工具，触发阈值就推送。
> 这部分的自动化，正是**第二步工程产品**可以做的（机会扫描/监控系统）。先手动+半自动跑通，再逐步自动化。

### 方法 3：渗透核心社区（②层的关键）
- 加入赛道的核心 Discord/Telegram/论坛，**不是潜水，而是观察"圈内人在讨论什么"**。
- 真正的早期信号往往在小圈子里先发酵，再扩散到媒体。
- 长期参与，建立存在感，慢慢能接触到更核心的信息流。

### 方法 4：建立"关键人物"清单
- 每个赛道，列出 10-30 个"消息最灵通/判断最准"的关键人物（创始人、核心开发者、资深玩家、靠谱分析师）。
- 集中追踪他们的公开发声。他们往往是①层信息的人肉节点。
- 长期目标：从"单向关注"到"双向交流"，进入信息互换的圈子（人脉本身是最强的一手源）。

### 方法 5：交叉验证 + 溯源
- 重要信息**至少两个独立源验证**，防假消息、防被人利用你来接盘。
- 拿到任何信息，问："它最早从哪来？为什么传到我这？传给我的人有什么动机？"
- 见 `01-认知基础/04` 的"二手信息陷阱"。

## 四、信息源的"分层订阅清单"模板

> 下面是框架，具体平台名你按赛道填入 `templates/信息源登记表.md`。这里给出**该订阅哪些"类型"的通道**。

### AI 工具/红利赛道（首选赛道，示例）
- **①官方一手**：主要大模型厂商的官方博客/发布页/官方社媒、主流 AI 产品的更新日志。
- **①代码/产品**：关键开源项目的 GitHub release、热门 AI 工具的版本更新。
- **②社区**：AI 开发者社区、产品发现社区（新品首发地）、相关 Discord/Telegram。
- **①关键人物**：AI 领域消息灵通的开发者/产品人/研究者。
- **③媒体**：高质量 AI newsletter、深度测评作者。
- **④聚合**：AI 工具榜单/导航/趋势页（用来兜底，不依赖）。

### 加密信息赛道（次选，严守合规，示例）
- **①官方一手**：关注的协议/项目官方频道、交易所公告页。
- **①链上数据**：链上浏览器、链上数据看板、钱包/资金流监控。
- **②社区**：核心 Discord/Telegram、专业论坛。
- **①关键人物**：靠谱的研究者/开发者（避开纯喊单网红）。
- **③媒体**：行业资讯站、研究机构报告。

> ⚠️ 加密赛道：所有信息源都用来"了解事实和风险"，**不是用来找喊单依据**。

## 五、信息源维护的纪律

信息源不是建一次就完事，要持续优化（配合知识库 `05-定期扫描SOP`）：

1. **每月体检**：哪些源持续给价值？哪些是噪音？砍掉噪音源。
2. **持续上移**：努力从③④层往①②层渗透。这是你信息优势的来源。
3. **记录"首发地"**：每次发现一个机会，回溯"它最早出现在哪"，把那个源加入清单。久而久之你会积累一张"哪里最早出消息"的地图——**这张地图本身就是护城河**。
4. **控制数量**：信息源宁缺毋滥。源太多会淹没你，反而错过重点。

## 六、这一切如何构成"信息套利"

回到你的理解，闭环是这样的：

```
你投入时间+人脉+工具，构建了①②层信息源网络（别人没有/嫌麻烦建）
        ↓
你比订阅者更早、更全地拿到一手信号（信息差产生）
        ↓
你用知识库的框架加工成判断（认知差叠加）
        ↓
订阅者付费购买"省时间+借用你的信息网络和判断"（价差变现）
        ↓
收入反哺你投入更多到信息源和工具（飞轮，护城河加深）
```

> **你的信息源网络 = 你的"生产资料"。** 别人能复制你的简报格式，但复制不了你长期积累的①②层信息源网络和"哪里最早出消息"的地图。这就是为什么这个生意越做越有壁垒。
